# History of C üï∞Ô∏è

C was developed at AT & T Bell Labs in **1972** by **Dennis Ritchie**.

# Character Set üî°

The basic building blocks in C are **tokens**, which are of six types:

1. Keywords (e.g., `int`, `while`) üóùÔ∏è
2. Identifiers (e.g., `main`, `total`) üè∑Ô∏è
3. Constants (e.g., `10`, `20`) üî¢
4. Strings (e.g., `"Shree Ram"`) "üí¨"
5. Special symbols (e.g., `[ ]`, `{ }`) üî£
6. Operators (e.g., `+`, `/`, `*`) ‚ûï‚ûó‚úñÔ∏è

## Keywords üîë

There are 32 keywords in C. They convey a special meaning to the C compiler and cannot be used as variable names.  
Examples: `continue`, `default`, `break`, `if`, `else`, `short`, etc. üö´üßæ

---

## Identifiers üè∑Ô∏è

Identifiers are user-defined names for variables, functions, and arrays.  
Rules:

1. Must begin with a letter or underscore (`_`) ‚ú®
2. Can only contain letters, digits, and underscores üî§
3. Cannot be a keyword üö´
4. Cannot contain whitespace (no spaces) ‚õî
5. Up to 31 characters long (only first 31 are significant) üìè

## Data Types in C üßÆ

C provides three types of data types:

1. **Primitive Data Types** ‚Äì built-in types (`int`, `char`, `float`, `double`, `void`) üîπ
2. **User Defined Data Types** ‚Äì defined by the user (`enum`, `struct`, `union`, `typedef`) üõ†Ô∏è
3. **Derived Data Types** ‚Äì derived from primitive types (`array`, `pointer`, `string`, `function`) üìö.

**Range formulas:**

- Signed: `-2^(n-1)` to `2^(n-1) - 1` ‚ûñ
- Unsigned: `0` to `2^n - 1` ‚ûï

---

## Typecasting üîÑ

### Implicit Typecasting üîÅ

Automatic conversion done by the compiler.

```cpp

char ch = 'a';
int res = 20 + 'a';
cout << res;
```

### Explicit Typecasting ‚úÇÔ∏è

Manually done by the programmer.

```cpp

num = (int) 4.534;
cout << num;
```

## Operators ‚ûó

Operators perform arithmetic or logical operations on operands. ‚öñÔ∏è

Operator precedence determines evaluation order.  
Associativity decides direction (left-to-right or right-to-left). ‚ÜîÔ∏è

### Operator Categories üßæ

| Level | Operators                                | Type                           | Associativity |
| ----- | ---------------------------------------- | ------------------------------ | ------------- | ------------- | ------------- |
| I     | `()`, `[]`, `->`, `.`                    | Function, Array, Member access | Left-to-Right |
| II    | `!`, `~`, `++`, `--`, `&`, `*`, `sizeof` | Unary                          | Right-to-Left |
| III   | `*`, `/`, `%`                            | Multiplicative                 | Left-to-Right |
| IV    | `+`, `-`                                 | Additive                       | Left-to-Right |
| VI    | `<`, `>`, `<=`, `>=`                     | Relational                     | Left-to-Right |
| VII   | `==`, `!=`                               | Equality                       | Left-to-Right |
| VIII  | `&`                                      | Bitwise AND                    | Left-to-Right |
| IX    | `^`                                      | Bitwise XOR                    | Left-to-Right |
| X     | `                                        | `                              | Bitwise OR    | Left-to-Right |
| XI    | `&&`                                     | Logical AND                    | Left-to-Right |
| XII   | `                                        |                                | `             | Logical OR    | Left-to-Right |
| XIII  | `?:`                                     | Ternary                        | Right-to-Left |
| XV    | `,`                                      | Comma                          | Left-to-Right |

## Types of Errors ‚ö†Ô∏è

1. **Syntax Error** ‚Äì Violates language rules (compile-time error). üõë
2. **Runtime Error** ‚Äì Occurs during execution (e.g., divide by zero). üí•
3. **Linker Error** ‚Äì Missing libraries or object files. üîó
4. **Logical Error** ‚Äì Incorrect program logic (runs but wrong output). üß©
5. **Semantic Error** ‚Äì Code is valid but meaningless (e.g., `int b = "om";`). ‚ùì

## Compile and Run C/C++ Code üõ†Ô∏è

> If not used, the default name is `a.out` or `a.exe`. ‚ö†Ô∏è
